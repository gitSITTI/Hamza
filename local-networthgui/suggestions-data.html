4 — MCP tools the server exposes
🔌

Property Tools

Address lookup, value refresh, portfolio read
  • GET
    Lookup property by address — returns Zestimate, assessed value, taxes, parcel ID/api/property/lookup?address=…
  • GET
    Get full property portfolio — all saved properties with latest values/api/property/portfolio
  • POST
    Add a property to your portfolio (saves to local DB)/api/property/add
  • PUT
    Update property details (balance, rent, mortgage rate)/api/property/:id
  • GET
    Refresh cached values from live sources (Zillow + assessor)/api/property/:id/refresh
🏛️

SOS / Entity Tools

MO + VA business entity & deed lookups
  • GET
    Look up MO SOS business entity by name or entity number/api/sos/mo/entity?name=…
  • GET
    Look up VA SCC entity — annual reports, registered agent, status/api/sos/va/entity?name=…
  • GET
    Search deed / title records by address (MO Recorder of Deeds)/api/deed/mo?address=…
  • GET
    Search VA Circuit Court land records by parcel / owner/api/deed/va?parcel=…
  • POST
    Save entity to local profile (LLC ownership, registered agent, state)/api/entity/save
📊

Market Data Tools

Rates, crypto, stocks — cached locally
  • GET
    Current 30-yr fixed mortgage rate (FRED, refreshes daily)/api/rates/mortgage
  • GET
    CPI / inflation rate — latest BLS release/api/rates/inflation
  • GET
    Crypto prices for your tracked coins/api/market/crypto?symbols=BTC,ETH,…
  • GET
    Stock / ETF quotes for your tracked positions/api/market/stocks?symbols=VOO,…
  • GET
    HUD fair-market rent for a zip code/api/rent/fmr?zip=63108
👤

Profile / Snapshot Tools

Store & restore all personal financial data
  • GET
    Get full current profile snapshot (all sliders, properties, market positions)/api/profile/current
  • POST
    Save profile snapshot — versioned with timestamp/api/profile/save
  • GET
    List all saved profile versions/api/profile/history
  • POST
    Restore profile from a past snapshot/api/profile/restore/:id
  • GET
    Export full data package (JSON + CSV) for backup/api/profile/export

address TEXT -- full street address
state TEXT -- MO | VA
type TEXT -- primary | rental | land
parcel_id TEXT -- county parcel number
zestimate REAL
assessed_valueREAL
annual_tax REAL
mortgage_balance REAL
mortgage_rate REAL
mortgage_term INTEGER -- years remaining
monthly_rent REAL -- 0 for primary
last_refreshedDATETIME
raw_zillow JSON -- full API response
raw_assessor JSON

market_positions table

-- Crypto, stocks, options, futures
id INTEGER PRIMARY KEY
asset_class TEXT -- crypto|stock|option|futures|commodity
symbol TEXT -- BTC, VOO, /ES, GLD…
balance_usd REAL -- current $ value
cost_basis REAL
annual_return_pct REAL -- assumption
annual_pl REAL -- for options/futures
last_price REAL
last_refreshed DATETIME

-- SOS entities (LLCs, corps)

6 — Build plan (step by step)
📦

Phase 1 — Local MCP server scaffold

~2–3 hours, Node.js + Express + SQLite
  1. Create mcp-server/ folderExpress app, SQLite via better-sqlite3, CORS enabled for localhost
  2. Define SQLite schemaRun migration on startup — creates all 4 tables if missing
  3. Wire profile endpointsGET/POST /api/profile — read and write the full app state JSON
  4. Add MCP tool manifestGET /mcp/tools — returns tool list so Claude can discover and call them
  5. Add to Claude Code MCP configPoint ~/.claude/settings.json mcpServers at http://localhost:3333/mcp
🏠

Phase 2 — Property address auto-fill

~3–4 hours, first real data integration
  1. Zillow / Rentcast API keyRegister at Rentcast (free tier) — returns Zestimate-equivalent + rent estimate
  2. MO county assessor fetchSt. Louis City and County have REST/GIS APIs — query by address, cache in properties table
  3. VA county assessor fetchFairfax + Arlington REST APIs — same pattern
  4. Address input widget in app.jsText input + "Fetch" button → calls localhost:3333/api/property/lookup → auto-fills sliders
  5. Cache + TTL logicSkip fetch if last_refreshed < 30 days ago; show "last updated" label in UI
📈

Phase 3 — Market data + SOS lookups

~2–3 hours, mostly free APIs
  1. FRED mortgage rate refreshDaily cron — MORTGAGE30US series, auto-updates rate slider default
  2. CoinGecko crypto pricesRefresh on demand; store in rate_cache; no API key needed for free tier
  3. MO SOS entity searchQuery businessfilings.sos.mo.gov API — save LLC details to entities table
  4. VA SCC entity searchQuery cis.scc.virginia.gov — same pattern
  5. Bulk refresh scriptnpm run refresh — hits all sources, updates all cached values, logs changes
🤖

Phase 4 — Claude MCP plugin wiring

~1–2 hours, makes Claude fully aware of your data
  1. Expose tool manifest at /mcp/toolsJSON array of tool names, descriptions, and input schemas — Claude discovers these on connect
  2. Handle tool calls at /mcp/callRoute each tool name to its handler function — returns structured JSON result
  3. Add server to Claude settings~/.claude/settings.json → mcpServers: { "nwgui-local": { type: "http", url: "http://localhost:3333/mcp" } }
  4. Test: ask Claude "what's my property portfolio worth today?"Claude calls get_portfolio → reads SQLite → answers with live data

Privacy note: The local MCP server runs entirely on your machine — no financial data, property values, or personal information ever leaves localhost unless you explicitly enable remote access. The SQLite database is stored at ~/.nwgui/data.db (or a path you choose) and is never synced to GitHub or cloud storage automatically.
