# Personal MCP — Technical Documentation

## Overview

The Personal MCP (`personal-mcp/`) is a Node.js + Express server that implements the Model Context Protocol (MCP) over HTTP. It serves as a personal financial brain: storing property, crypto, and investment data, providing market data from external APIs, and maintaining a full-text-searchable knowledge base (RAG) that Claude can query during any session.

**Port:** 3333  
**Protocol:** JSON-RPC 2.0 over HTTP POST /mcp  
**Auth:** `x-api-key` header or `Authorization: Bearer <token>`  
**Runtime:** Node.js 22 (built-in `node:sqlite` — no native compilation)

## Architecture

### Directory Structure

```
personal-mcp/
├── server.js          Entry point — Express app, routes, port binding
├── mcp.js             JSON-RPC 2.0 dispatcher (initialize, tools/list, tools/call)
├── db.js              SQLite initialization, schema migrations, encrypt/decrypt
├── auth.js            API key middleware + audit log writer
├── .env               Secrets (not committed — generated by setup.js)
├── .env.example       Template for new installs
├── plugins/
│   ├── index.js       Plugin registry — combines all tool arrays
│   ├── identity.js    6 tools: profile, snapshots, obligations
│   ├── property.js    5 tools: property lookup, investment analysis, vault
│   ├── market.js      6 tools: FRED rates, Polygon stocks, HUD rents
│   ├── crypto.js      5 tools: CoinGecko prices, portfolio, Roth, RE vs BTC
│   ├── projects.js    6 tools: project tracking, session context, goals
│   └── rag.js         6 tools: FTS5 knowledge base, Notion sync, context injection
└── scripts/
    └── setup.js       First-run: generates API key, DB secret, seeds initial project
```

### Request Flow

```
Claude Code / NetWorth GUI / curl
    |
    POST /mcp  { "jsonrpc": "2.0", "method": "tools/call", "id": 1, "params": {...} }
    |
    authenticateKey (auth.js) → validates x-api-key header
    |
    handleMCP (mcp.js) → dispatch()
    |
    switch(method):
      "initialize"   → return server info + capabilities
      "tools/list"   → return all tool schemas from plugins
      "tools/call"   → plugins.getTool(name).handler(args)
    |
    logToolCall (auth.js) → writes to audit_log table
    |
    Response: { "jsonrpc": "2.0", "id": 1, "result": { "content": [{"type": "text", "text": "..."}] } }
```

### HTTP Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | /health | None | Server status, uptime, table count |
| POST | /mcp | Required | Main MCP endpoint |
| GET | /tools | Required | Convenience: list all tools as JSON |

## Authentication and Security

### API Key

Generated by `scripts/setup.js` as a cryptographically random hex string prefixed with `pmcp-`. Format: `pmcp-<48 hex chars>`.

Authentication check in `auth.js`:
```javascript
const key = req.headers['x-api-key'] 
         || req.headers['authorization']?.replace('Bearer ', '');
if (!key || key !== process.env.MCP_API_KEY) {
  return res.status(401).json({ error: 'Invalid or missing API key' });
}
```

### Field-Level Encryption

Sensitive values (account balances, cost basis) are encrypted before storage using AES-256-CBC with a random IV per encryption operation.

`encrypt(text)` in `db.js`:
```
iv = 16 random bytes
key = scrypt(DB_SECRET, 'pmcp-salt', 32)
ciphertext = AES-256-CBC(key, iv).encrypt(text)
stored as: iv_hex + ':' + ciphertext_hex
```

`decrypt(blob)` reverses this. If the blob doesn't contain ':', it's returned as-is (unencrypted or null).

The DB_SECRET is generated by `setup.js` and stored in `.env` only. It never appears in code.

### Audit Log

Every `tools/call` invocation is logged to the `audit_log` table:
- `caller` — value of `x-caller` header (set by Claude Code or the calling app)
- `tool` — tool name
- `args` — JSON-stringified arguments (first 500 chars)
- `result` — JSON-stringified result (first 500 chars)
- `ts` — timestamp

## Database Schema

SQLite database at `personal-mcp.db` (WAL mode, foreign keys enabled).

### Tables (19 total)

#### `profile`
Singleton row (id=1). Stores owner name, email, and arbitrary metadata JSON.
```sql
id INTEGER PRIMARY KEY CHECK (id = 1)
name TEXT, email TEXT, meta TEXT DEFAULT '{}'
```

#### `snapshots`
Historical net worth snapshots. Each row is a point-in-time capture.
```sql
id TEXT PRIMARY KEY
label TEXT                      -- e.g. "June 2026 snapshot"
net_worth REAL
data TEXT                       -- full JSON of all slider values at snapshot time
created_at TEXT
```

#### `properties`
Property vault — owned, pipeline, and sold properties.
```sql
id TEXT PRIMARY KEY
nickname TEXT, address TEXT, type TEXT, status TEXT
purchase_price REAL, current_value REAL
mortgage_balance REAL, mortgage_rate REAL, mortgage_term INTEGER
monthly_rent REAL, monthly_expenses REAL, unit_count INTEGER
assessor_data TEXT, rentcast_data TEXT, notes TEXT
created_at TEXT, updated_at TEXT
```

#### `accounts`
Financial accounts. Balance field is AES-256 encrypted.
```sql
id TEXT PRIMARY KEY
type TEXT                       -- roth, ira, taxable, 401k, other
institution TEXT, nickname TEXT
balance TEXT                    -- ENCRYPTED
rate REAL, notes TEXT, updated_at TEXT
```

#### `market_positions`
Individual positions within accounts. cost_basis is encrypted.
```sql
id TEXT PRIMARY KEY
account_id TEXT                 -- FK to accounts
ticker TEXT, asset_type TEXT   -- crypto, stock, bond, etc.
quantity REAL
cost_basis TEXT                 -- ENCRYPTED
notes TEXT, updated_at TEXT
```

#### `entities`
Business entities (LLCs, corporations) the owner is associated with.
```sql
id TEXT PRIMARY KEY
name TEXT, type TEXT, state TEXT, sos_id TEXT
role TEXT                       -- owner, member, partner
ownership_pct REAL, equity_value REAL, annual_revenue REAL
notes TEXT, sos_data TEXT, created_at TEXT
```

#### `documents`
Document vault for tax returns, contracts, etc. (PDF parsing in Phase 3).
```sql
id TEXT PRIMARY KEY
category TEXT, year INTEGER
filename TEXT, parsed_data TEXT, summary TEXT
encrypted INTEGER DEFAULT 1
created_at TEXT
```

#### `projects`
Project tracking. Each project has a `context_block` — a text block Claude can inject at session start to resume work without re-explaining.
```sql
id TEXT PRIMARY KEY
name TEXT NOT NULL, description TEXT
status TEXT DEFAULT 'active'    -- active, planned, paused, complete
stack TEXT, repo TEXT, branch TEXT
progress_pct INTEGER DEFAULT 0
context_block TEXT              -- What Claude needs to know to work on this
notes TEXT, updated_at TEXT
```

#### `goals`
Financial and life goals with target values and dates.
```sql
id TEXT PRIMARY KEY
title TEXT, category TEXT       -- financial, property, business, crypto, retirement, lifestyle
target_value REAL, current_value REAL
target_date TEXT                -- YYYY-MM-DD
notes TEXT, updated_at TEXT
```

#### `obligations`
Recurring financial obligations: child support, alimony, loans.
```sql
id TEXT PRIMARY KEY
type TEXT                       -- child_support, alimony, loan, other
counterparty TEXT, monthly_amount REAL
start_date TEXT, end_date TEXT
notes TEXT
```

#### `rate_cache`
TTL-based cache for external API responses.
```sql
key TEXT PRIMARY KEY            -- e.g. "fred:MORTGAGE30US", "polygon:SPY"
value TEXT                      -- JSON-encoded cached value
fetched_at TEXT
ttl_seconds INTEGER DEFAULT 3600
```

#### `audit_log`
Append-only log of every tool call.
```sql
id INTEGER PRIMARY KEY AUTOINCREMENT
caller TEXT, tool TEXT, args TEXT, result TEXT
ts TEXT DEFAULT (datetime('now'))
```

#### `rag_documents`
Structured document store for the knowledge base.
```sql
id TEXT PRIMARY KEY
category TEXT, title TEXT, content TEXT, metadata TEXT, source TEXT
created_at TEXT, updated_at TEXT
```

#### `rag_fts` (FTS5 virtual table)
Full-text search index over rag_documents. Uses porter stemmer for English.
```sql
CREATE VIRTUAL TABLE rag_fts USING fts5(
  doc_id, category, title, content,
  tokenize='porter ascii'
)
```

### Schema Migrations

All schema lives in the `migrate()` function in `db.js`. It uses `CREATE TABLE IF NOT EXISTS` so it is safe to run on every server start. Adding a new table: add a `CREATE TABLE IF NOT EXISTS` block to `migrate()`.

## RAG Knowledge Base

### How Indexing Works

The `indexDoc()` function in `rag.js` is the internal indexing API (not exposed as a tool — called by other tools):

```javascript
async function indexDoc({ id, category, title, content, source, metadata })
```

1. Checks if document with this `id` already exists in `rag_documents`
2. If it exists: updates `rag_documents` and deletes the old FTS entry using `INSERT INTO rag_fts(rag_fts, doc_id) VALUES ('delete', ?)` (FTS5 delete syntax)
3. Inserts into `rag_documents`
4. Inserts into `rag_fts` with doc_id, category, title, content

Automatic indexing happens when:
- `save_snapshot` is called — snapshot content is indexed under category `finance`
- `save_property` is called — property data is indexed under category `property`
- `save_project` is called — project name + context_block indexed under category `project`

### How Search Works

`search_knowledge` tool:

1. Sanitizes query: strips `'"*():` characters, appends `*` for prefix matching
2. Runs FTS5 MATCH query:
   ```sql
   SELECT doc_id, category, title, snippet(...) as excerpt
   FROM rag_fts
   WHERE rag_fts MATCH ?
   ORDER BY rank
   LIMIT ?
   ```
3. The `rank` column is FTS5's built-in BM25 relevance score
4. `snippet()` function extracts the most relevant portion of content with `...` ellipsis
5. Enriches each result with source and created_at from `rag_documents`
6. Falls back to `LIKE` search if FTS5 throws (e.g., invalid query syntax)

`get_context_for_session` tool uses the same FTS search but returns the full content formatted as injection blocks:
```
[CATEGORY] Title
Full content

---

[CATEGORY] Title
Full content
```

## Plugin Reference

### identity.js — 6 tools

**get_profile** — Returns profile row, latest snapshot, property count, account count, active project count. No inputs.

**update_profile** — Updates name, email, or meta JSON on the singleton profile row. Inputs: `name` (string), `email` (string), `meta` (object).

**save_snapshot** — Saves a net worth snapshot to `snapshots` table and indexes it in RAG. Inputs: `label` (string), `net_worth` (number, required), `data` (object — full slider state).

**get_snapshots** — Returns historical snapshots ordered by date, newest first. Computes change over period if 2+ snapshots exist. Inputs: `limit` (number, default 12).

**list_obligations** — Returns all obligations with total monthly obligation sum. No inputs.

**save_obligation** — Creates a new obligation record. Inputs: `type` (required), `counterparty`, `monthly_amount` (required), `start_date`, `end_date`, `notes`.

---

### property.js — 5 tools

**lookup_property** — Fetches property data from Rentcast API by address. Returns value estimate, rent estimate, bedroom/bathroom count, sqft, year built. Requires `RENTCAST_API_KEY` in `.env`. Input: `address` (required).

**analyze_investment** — Full investment analysis from inputs. Computes: cap rate, cash-on-cash return, DSCR, GRM, 10-year IRR (Newton-Raphson), and a verdict (STRONG BUY / BUY / HOLD-NEGOTIATE / PASS). All inputs have sensible defaults. Required inputs: `purchase_price`, `monthly_rent`.

Verdict thresholds:
- STRONG BUY: cap rate >= 6%, CoC >= 8%, DSCR >= 1.2
- BUY: cap rate >= 5%, CoC >= 6%, DSCR >= 1.1
- HOLD/NEGOTIATE: cap rate >= 4%, DSCR >= 1.0
- PASS: below thresholds

**save_property** — Saves property to vault. Status options: `owned`, `pipeline`, `sold`. Also indexes in RAG. Required: `address`.

**list_properties** — Lists properties by status with computed equity (current_value - mortgage_balance) and monthly cash flow. Returns total equity across portfolio. Input: `status` (default: "owned").

**property_stress_test** — Runs 5 stress scenarios on a property: base case, 15% vacancy, rate +2%, combined, and 20% price drop at exit. Required: `purchase_price`, `monthly_rent`, `loan_amount`.

---

### market.js — 6 tools

All external API responses are cached in `rate_cache` table with TTL. Default TTL: 1 hour for FRED and Polygon, 1 week for HUD.

**get_mortgage_rates** — FRED series MORTGAGE30US and MORTGAGE15US. Returns current 30yr and 15yr fixed rates. Requires `FRED_API_KEY`.

**get_stock_quote** — Polygon.io previous-day close for a single ticker. Returns close, open, volume, date. Requires `POLYGON_API_KEY`. Input: `ticker`.

**get_multiple_quotes** — Batch version of get_stock_quote. Input: `tickers` (array of strings).

**get_rent_market** — HUD Fair Market Rents for a state. Useful for benchmarking rental income against government data. Requires `HUD_API_KEY`. Input: `state` (2-letter code).

**get_economic_indicator** — Any FRED series by ID. Examples: FEDFUNDS (fed funds rate), UNRATE (unemployment), CPIAUCSL (CPI), HOUST (housing starts). Requires `FRED_API_KEY`. Input: `series_id`.

**compare_investments** — Compares annualized CAGR across multiple investments from cost/current value/hold period. Sorts by CAGR descending. Input: `investments` array with `name`, `cost`, `current_value`, `years_held`, optional `annual_income`.

---

### crypto.js — 5 tools

Supported tickers (mapped to CoinGecko IDs): BTC, ETH, SOL, BNB, ADA, AVAX, MATIC, LINK, DOT. Free tier works without API key; set `COINGECKO_API_KEY` for Pro tier. Cache TTL: 5 minutes.

**get_crypto_prices** — Live prices with 24h change and market cap. Input: `tickers` (array, default: ["BTC","ETH","SOL"]).

**get_portfolio_value** — Looks up all saved crypto positions (`market_positions` where `asset_type = 'crypto'`), fetches live prices, returns current value per position and total. No inputs.

**save_crypto_position** — Saves a holding to `market_positions`. Creates an account record if none exists for the institution+type combination. Cost basis is AES-256 encrypted. Required: `ticker`, `quantity`. Optional: `cost_basis`, `account_type` (roth, ira, taxable, 401k, other), `institution`.

**roth_crypto_analysis** — Models tax-free growth in Roth IRA vs taxable account. Returns: future value, Roth keeps (full amount), taxable keeps (after capital gains tax), dollar advantage of Roth. Required: `amount_usd`, `growth_rate`, `years`. Optional: `ticker`, `tax_rate` (default 15%).

**re_vs_btc_offset** — Models the tradeoff of redirecting down payment dollars to BTC Roth IRA instead of real estate equity. Accounts for foregone RE appreciation and optional HELOC interest cost. Returns winner over the specified period. Required: `redirect_amount`, `property_value`, `btc_growth_rate`, `years`. Optional: `property_appreciation` (default 3%), `heloc_rate`.

---

### projects.js — 6 tools

**list_projects** — Lists all projects with status, progress percentage, repo, and branch. Input: `status` (optional filter).

**get_project** — Returns full project record including `context_block`. Supports partial name match. Input: `name_or_id` (required).

**save_project** — Creates or updates a project. Pass `id` to update. If creating, indexes in RAG. The `context_block` field is the most important — it's what Claude injects at session start to resume work. Required: `name`.

**get_session_context** — Returns combined context block for all active projects + latest net worth snapshot + obligations + properties + goals. Format is designed for injection at the start of a Claude session. No inputs.

**save_goal** — Creates a goal with title, category, target value, current value, and target date. Required: `title`.

**gap_analysis** — Returns all goals with current vs target, gap in dollars, percent complete, days remaining, and on-track status. Uses latest snapshot net worth as current value if goal doesn't have its own `current_value`. No inputs.

---

### rag.js — 6 tools

**search_knowledge** — Primary RAG search. FTS5 full-text with BM25 ranking, category filter, snippet extraction. Falls back to LIKE search. Required: `query`. Optional: `category`, `limit` (default 10).

**get_document** — Returns full document content by ID. Input: `id` (required).

**index_note** — Manually add any text to the knowledge base. Good for plans, meeting notes, decisions. Required: `content`. Optional: `title`, `category`, `source`.

**list_knowledge_index** — Shows document counts by category and last update time. No inputs.

**sync_notion** — Pulls all accessible Notion pages into the knowledge base. Rate-limited to once per 15 minutes (pass `force_refresh: true` to override). Requires `NOTION_API_KEY`. Currently indexes page title, URL, last edit time — full block content sync is Phase 3.

**get_context_for_session** — Retrieves the most relevant documents for a topic, formatted as injection blocks. Required: `topic`. Optional: `max_docs` (default 5).

## Adding a New Tool

1. Open the appropriate plugin file in `plugins/`
2. Add a new object to the `tools` array:
   ```javascript
   {
     name: 'tool_name',
     description: 'What this tool does — shown to Claude',
     inputSchema: {
       type: 'object',
       properties: {
         param1: { type: 'string', description: 'What it is' }
       },
       required: ['param1']
     },
     async handler({ param1 }) {
       const db = getDb();
       // ... do work
       return { result: 'value' };
     }
   }
   ```
3. Restart the server — `plugins/index.js` auto-discovers all tools via spread into `allTools`
4. Test: `curl -X POST http://localhost:3333/mcp -H "x-api-key: YOUR_KEY" -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'`
